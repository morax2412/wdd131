Website Planning Document

Project Topic: Professional Psychologist Website

This site plan is the blueprint for the final WDD 131 project website. It defines the site name, purpose, target audience scenarios, design system (colors + typography), site structure, required JavaScript behaviors, and wireframes for mobile and desktop.

1. Site Name

Proposed Name

MindCare Psychology (or MindCare Psicología)

I chose this name because it communicates emotional support, professionalism, and a calm, safe space for clients.

Psychologist Name (Client)

Lic. Daniela Estefany Rondan Suazo

The site is designed to support a psychologist’s professional presence and help potential clients request appointments.

2. Site Purpose

The purpose of this website is to present a professional profile for a psychologist, explain services, build trust with visitors, and make it easy to request an appointment using a contact form. The site will contain service descriptions, a professional bio, a simple contact process, and a references page.

Primary Goals

  • Provide clear information about the psychologist and the services offered.
  • Build credibility through professional layout, tone, and organization.
  • Help users choose the right service by category.
  • Allow users to submit an appointment request via a valid HTML form.

3. Target Audience & Scenarios

Typical visitor questions:

  • “Do you help with anxiety, stress, or relationship problems?”
  • “What services do you offer and how do sessions work?”
  • “How can I request an appointment?”
  • “Where are you located and do you offer online sessions?”

4. Color Scheme

Calm and trustworthy healthcare palette. Primary supports headers and CTAs, and Accent supports highlights.

Primary: Deep Teal #2E6F6B

Use: header, headings, buttons, links, key highlights.

Accent: Warm Gold #F4B860

Use: call-to-action accents, badges, small highlights.

Support Colors

  • Background: #F6F8FA
  • Body text: #1F2937
  • Borders: #E5E7EB

5. Typography

  • Headings: Poppins (600–700)
  • Body: Inter (400–600)

6. Site Map (Pages & Content Plan)

  • index.html (Home): hero introduction, services preview, about preview, CTA to contact.
  • page2.html (Services): full services list + filter by category (JS dynamic rendering).
  • page3.html (About): professional bio, approach, specialties, values.
  • contact.html (Contact): valid HTML form + confirmation message.
  • references.html: citations for images/fonts/resources (unstyled allowed).

7. JavaScript Plan (W06 Requirements)

This website will demonstrate the required JavaScript behaviors:

  • More than one function: render services, filter services, update footer dates, handle form submission.
  • DOM interaction: select elements and inject service cards and messages dynamically.
  • Events: filter change event (Services page) and form submit event (Contact page).
  • Conditional branching: validate form fields and filter category selection.
  • Objects & Arrays: psychologist profile object + services array of objects.
  • Array methods: filter() and map() to generate HTML.
  • Template literals: used for all dynamic output strings.
  • localStorage: store user name/email on contact submit and remember filter selection on services page.

10. Notes for Development

  • Responsive design using mobile-first CSS and media queries.
  • Accessibility: semantic HTML, headings, labels, focus states, and contrast checks.
  • Performance: optimized images + lazy loading.
  • Validation: no HTML/CSS errors and no JS console errors.
